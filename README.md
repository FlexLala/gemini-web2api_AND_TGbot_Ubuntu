# Gemini Telegram Bot + gemini-web2api

Полное решение: локальный **gemini-web2api** (запросы идут через ваш сервер в Германии) + **Telegram-бот** с доступом, статистикой, LaTeX-защитой и инлайн-режимом.

## Что внутри

| Компонент | Описание |
|-----------|----------|
| `gemini-web2api` | Прокси Google Gemini Web → OpenAI API. Работает без авторизации Google. Запросы к Gemini идут с немецкого IP вашего сервера. |
| `telegram-bot` | Бот на `python-telegram-bot`. История диалогов, выбор моделей, админ-панель, инлайн-режим. |

## Требования

- Сервер с Docker + Docker Compose (plugin)
- Git
- Свободные порты внутри Docker-сети (внешние 443/8443/1443 **не нужны**)
- Telegram Bot Token (от [@BotFather](https://t.me/BotFather))
- Ваш Telegram User ID (от [@userinfobot](https://t.me/userinfobot))

## Быстрый старт

```bash
git clone <этот-репозиторий> gemini-telegram-bot
cd gemini-telegram-bot
chmod +x setup.sh
./setup.sh
```

Скрипт:
1. Проверит `git`, `docker`, `docker compose`
2. Склонирует `gemini-web2api`
3. **Интерактивно** спросит токены и настройки
4. Создаст `.env` и `config.json` через **EOF** (никакого nano)
5. Запустит контейнеры

## Команды бота

**Для всех:**
- `/start`, `/help` — справка
- `/model` — выбрать модель (Flash, Thinking, Pro, Auto, Lite)
- `/clear` — очистить историю диалога
- Просто текст — диалог с Gemini

**Инлайн:**
- `@YourBot вопрос` — мгновенный ответ (только экономная модель, без истории)

**Админ:**
- `/stats` — токены и запросы по пользователям
- `/users` — список пользователей
- `/allow <id>` — одобрить доступ
- `/block <id>` — заблокировать

## Особенности

### 1. Доступ по заявке
По умолчанию бот работает только у вас. Если кто-то другой напишет — вы получите уведомление с кнопками:
- **Разрешить** — пользователь сразу получает доступ
- **Игнорировать 5 мин** — повторные запросы от этого ID не будут приходить 5 минут

### 2. LaTeX не ломает сообщения
Gemini любит выдавать формулы. Бот:
- Экранирует HTML-сущности (`<`, `>`, `&`)
- Оборачивает `\(...\)` и `$...$` в `<code>`
- Оборачивает `\[...\]` и `$$...$$` в `<pre>`
- Разбивает длинные ответы на части (лимит 4000 символов)

### 3. Рассуждения в свёрнутом блоке
Если модель выдаёт `<thinking>...</thinking>`, бот автоматически помещает это в `<blockquote expandable>` (сворачиваемая цитата Telegram), а ответ показывает отдельно.

### 4. Ошибки сразу видны
Любые `502 Bad Gateway`, таймауты или ошибки upstream — бот мгновенно пишет пользователю понятное сообщение.

### 5. Конфиденциальность
Админ видит **только**:
- ID пользователя
- Количество запросов
- Примерное количество токенов

**Не видит** содержимое сообщений.

## Управление

```bash
# Логи бота
docker compose logs -f telegram-bot

# Логи API
docker compose logs -f gemini-api

# Перезапуск
docker compose restart

# Полная остановка
docker compose down

# Обновление gemini-web2api
cd gemini-web2api && git pull && cd .. && docker compose up --build -d
```

## Безопасность

- `.env` и `config.json` содержат ключи. **Не коммитьте их**.
- API-key для gemini-web2api задан вами и известен только боту (внутри Docker-сети).
- Внешний порт gemini-web2api **не проброшен** наружу — доступ только из контейнера бота.

## Если что-то пошло не так

1. Проверьте логи: `docker compose logs -f telegram-bot`
2. Убедитесь, что `ADMIN_ID` — это именно ваш числовой ID (не юзернейм)
3. Убедитесь, что у бота в @BotFather включён **Inline Mode**
4. Если gemini-web2api пишет `BardErrorInfo [1060]` — попробуйте перезапустить: `docker compose restart gemini-api`

## Идеи для доработки

- Прикрутить **Redis** для хранения истории вместо SQLite (если пользователей много)
- Добавить **голосовые сообщения** (Speech-to-Text через Whisper или Google STT)
- **Квоты**: ограничить количество запросов в сутки для обычных пользователей
- **Проксируемый поток** (streaming) — сейчас бот ждёт полного ответа, можно сделать live-печать
